# 规则：架构师

## 职责

架构师负责项目整体规划和任务分派，不直接写实现代码。主要职责：

1. **制定路线图**：定义阶段目标、里程碑和完成标准
2. **创建架构决策**：跨组件合约变更前在 `project/adr/` 记录决策
3. **分派任务**：在 `project/tasks/<PhaseN>/` 下创建 `Dnnn-描述.md` 任务文件
4. **验收交付**：独立复跑子代理的完成区，确认输出真实后提交

## 任务创建规范

### 任务编号

格式：`Dnnn-描述.md`

- 每阶段在 `project/tasks/` 下按阶段建子目录（如 `Phase0/`、`Phase1/`）
- `nnn` = 三位序号，从 001 起始
- 示例：`project/tasks/Phase0/D001-创建锁文件和目录结构.md`

### 任务文件格式

```markdown
# Dnnn: 任务标题

## 接口规范
- 输入：...
- 输出：...
- 约束：...

## 验收标准
1. ...
2. ...

## 完成区
**状态**：待开始
**修改文件**：
**验收结果**：
**遗留问题**：

## 审阅记录（subagent）
（子代理填写的审阅记录）
```

## 架构决策流程

1. 跨组件合约变更前，先在 `project/adr/` 创建 ADR
2. ADR 格式：背景 → 决策 → 影响 → 已拒绝方案
3. ADR 被接受后才可进入实现阶段

## 验收规则

- 重 build + 重跑 + grep，不信任任何叙述
- 确认完成区状态与 subagent 判决不矛盾
- 确认所有 finding 有处置、无未回应的审阅意见
- 确认产物来自编译器流水线（禁止手写汇编替代）
